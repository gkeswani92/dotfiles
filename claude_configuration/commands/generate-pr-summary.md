# Generate PR Summary

Generate a PR summary based on our conversation context using the following template:

### What has changed?
- Summarize the technical changes made in this PR based on our conversation
- List specific files, components, or features modified
- Keep it factual and concise

### Why this change?
- Explain the business/technical motivation for the change
- Describe the problem being solved or improvement being made
- Clarify the impact or benefits of this change

### How can this change be tested?
- If I haven't provided testing instructions, leave this section blank with just the heading

After generating the summary:
1. Print it to console
2. Copy it to clipboard if possible
3. Ask me: "Is there a GitHub issue or other reference for this change?"
4. Ask me: "How can this change be tested?" (if not already covered in our conversation)
5. Update the summary with my answers and provide the final version

Keep the tone concise throughout.
